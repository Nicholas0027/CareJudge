\begin{abstract}
Large language models are widely used as automated judges, yet they are overconfident and sensitive to how an evaluation protocol is worded. We ask whether a judge's \emph{protocol (in)stability}---whether its verdict survives equivalent rubric paraphrases, response-order swaps, and resampling---is a reliable per-instance error signal. We find that it is, but only conditionally: the signal is \textbf{competence-gated}. Above a judge-accuracy threshold, unstable verdicts are markedly more error-prone than stable ones. Below it, the signal inverts: a near-random judge is \emph{stably wrong}, so stability no longer implies correctness. A negative control isolates this effect, and a mechanistic decomposition of stable-but-wrong errors explains it.

We operationalize the finding as \textbf{CARE-Judge} (Calibrated, Abstaining, and Robust Evaluation). It fuses the three stability families into a learned correctness calibrator and accepts or abstains under an explicit risk budget certified by an exact Clopper--Pearson threshold. We evaluate five judges spanning a controlled Qwen2.5 size ladder and two API models across four benchmarks. Fusion improves the risk--coverage tradeoff most in the mid-accuracy band, ties an already-saturated frontier judge, and abstains where a judge is near-chance---the pattern competence-gating predicts. The result is a selective evaluator that reports when it should not be trusted, and a characterization of the regime in which protocol stability is a valid reliability signal.
\end{abstract}
