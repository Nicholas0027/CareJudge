\begin{abstract}
Large language models are widely used as automated judges, yet they are overconfident and sensitive to how an evaluation protocol is worded. We study whether a judge's \emph{protocol (in)stability} is a reliable per-instance error signal. A judge is protocol-stable on an item if its verdict survives equivalent rubric paraphrases, response-order swaps, and resampling. We find that instability signals error, but only under a condition we call \textbf{competence-gating}. When a judge is accurate enough on a task, unstable verdicts are far more likely to be wrong than stable ones. When it is near random, the signal reverses: the judge becomes \emph{stably wrong}, so stability no longer implies correctness. A negative control and a decomposition of the stable-but-wrong errors confirm this reading.

We build \textbf{CARE-Judge} (Calibrated, Abstaining, and Robust Evaluation) on this finding. It fuses the three stability families into a learned correctness calibrator, then accepts or abstains under an explicit risk budget set by an exact Clopper--Pearson threshold. We test five judges, spanning a controlled Qwen2.5 size ladder and two API models, on four benchmarks. Fusion improves the risk--coverage tradeoff most in the mid-accuracy band, matches an already saturated frontier judge, and abstains when a judge is near chance. This is the pattern competence-gating predicts. CARE-Judge is thus a selective evaluator that signals when it should not be trusted, together with a clear account of the regime in which protocol stability is a valid reliability signal.
\end{abstract}
